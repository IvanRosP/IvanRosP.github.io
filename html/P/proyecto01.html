PROYECTO 01

Vulneración de Máquina Virtual SnakeOil

Iván Ros Padilla

1. Introducción

Dentro del área de la seguridad informática, las pruebas de penetración permiten identificar vulnerabilidades presentes en sistemas antes de que puedan ser explotadas por actores maliciosos. Para desarrollar habilidades prácticas en este campo, existen plataformas como VulnHub que proporcionan máquinas virtuales intencionalmente vulnerables diseñadas para practicar técnicas de análisis y explotación.

En este proyecto se realizó el proceso de análisis y explotación de la máquina virtual SnakeOil, la cual contiene múltiples vulnerabilidades que permiten obtener acceso al sistema mediante técnicas de enumeración, explotación y escalamiento de privilegios.

2. Desarrollo Técnico

2.1 Preparación del Entorno

Para realizar la práctica se configuró un entorno virtual utilizando dos máquinas virtuales:

  • Máquina atacante: Kali Linux.
  • Máquina vulnerable: SnakeOil.
  • Configuración de red: NAT privada para comunicación directa.

La configuración de red permitió que ambas máquinas se comunicaran dentro de una red aislada, evitando interferencias con otras redes externas.

2.2 Reconocimiento y Enumeración

El primer paso del proceso consistió en identificar los servicios activos en la máquina vulnerable. Para ello se realizaron técnicas de reconocimiento de red con herramientas como:

  • Escaneo de puertos.
  • Identificación de servicios web.
  • Búsqueda de directorios ocultos.

Durante esta fase se detectó un servidor web activo dentro de la máquina vulnerable, el cual contenía diferentes directorios accesibles públicamente. Estos directorios permitieron identificar posibles puntos de entrada al sistema.

2.3 Acceso Inicial

Mediante la exploración de los directorios del servidor web se logró encontrar información que permitió obtener acceso a una shell dentro del sistema. Este acceso inicial permitió ejecutar comandos dentro del sistema operativo de la máquina vulnerable.

2.4 Escalamiento de Privilegios

Una vez dentro del sistema, se realizaron técnicas de enumeración local para identificar configuraciones incorrectas o vulnerabilidades que permitieran elevar los privilegios del usuario actual.

Finalmente se logró obtener acceso a la cuenta root, lo cual representa el nivel máximo de privilegios dentro del sistema operativo.

4. Reflexión Técnica

Este proyecto permitió comprender de manera práctica cómo se desarrollan las pruebas de penetración en entornos controlados. A través de la exploración de servicios, enumeración de directorios y explotación de vulnerabilidades, se puede observar cómo un sistema mal configurado puede ser comprometido completamente.

Las organizaciones deben aplicar medidas de seguridad como actualizaciones constantes, segmentación de red, monitoreo de actividad y auditorías de seguridad periódicas para reducir el riesgo de ataques reales.
