ACTIVIDAD 03

Interpretación y traducción de políticas de filtrado en iptables

Ivan Ros Padilla

1. Introducción

iptables es una herramienta de administración de firewall en sistemas Linux que permite definir reglas de filtrado de paquetes basadas en tablas, cadenas y políticas. Esta actividad analiza la interpretación de reglas y su traducción a escenarios prácticos de control de tráfico en redes.

2. Desarrollo Técnico

2.1 Flujo del paquete en iptables

Cuando un paquete llega al sistema, atraviesa diferentes tablas según su propósito:

  • RAW – Control previo al seguimiento de conexión.
  • MANGLE – Modificación de encabezados.
  • NAT – Traducción de direcciones.
  • FILTER – Filtrado principal (INPUT, OUTPUT, FORWARD).

2.2 Ejemplos de reglas interpretadas

Regla Interpretación
iptables -P INPUT DROP Establece política por defecto para bloquear tráfico entrante.
iptables -A INPUT -p tcp --dport 80 -j ACCEPT Permite tráfico HTTP entrante.
iptables -A OUTPUT -j ACCEPT Permite todo el tráfico saliente.
iptables -A INPUT -s 192.168.1.50 -p tcp --dport 22 -j ACCEPT Permite acceso SSH solo desde una IP específica.
-m state --state NEW,ESTABLISHED Permite conexiones nuevas y ya establecidas.

2.3 Aplicación en escenarios reales

Las reglas de iptables permiten implementar políticas restrictivas como:

  • Modelo de seguridad por defecto “deny all”.
  • Segmentación lógica de servicios.
  • Protección contra accesos no autorizados.
  • Control granular por puerto, protocolo e IP.

4. Reflexión Técnica

La correcta interpretación de reglas en iptables es fundamental para garantizar una política de seguridad coherente. Una configuración incorrecta puede generar brechas de seguridad o bloqueos involuntarios de servicios críticos.

La implementación de políticas restrictivas por defecto y la autorización explícita de servicios necesarios representa una buena práctica en la administración de firewalls.
